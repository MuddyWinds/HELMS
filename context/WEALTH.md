## Purpose

Operational ledger for the external track — what the user actually owns, what is compounding, what is being built, and what has been killed. The companion file to `context/GOALS.md` (north star) and `context/PLAN.md` (strategy). Methodology for how to evaluate, build, and sell lives in `protocols/CAPITALIST.md`.

The distinction that justifies a separate file: **savings rate × salary is a linear track. Wealth is a non-linear track.** A single metric (war chest balance) cannot surface the difference between labour-assets and ownable-assets. This file separates them.
